1. Building the render only sample
  a. Install VS 2015
  b. Install Windows 10 WDK from https://msdn.microsoft.com/en-us/windows/hardware/dn913721.aspx

2. Running the render only sample in a VM.

To run the sample within a VM, you will need to modify the ACPI information to report the existance of
a render only device.  Here are the steps you need to follow to modify the ACIP information:
  a. launch an administrator prompt within the VM
  b. turn on test signing
    > bcdeit -set testsigning on
  c. get a copy of the asl.exe tool (you can pull this from the bin\idw directory of a build, from MSDN or from WDK\Tools\x64\ACPIVerify)
  d. Use asl.exe to get a copy of the DSDT.ASL source (this the ACPI file used to report devices)
    > asl.exe /tab=DSDT
  e. Modify the definition block in the DSDT.ASL bumping up version of the DSDT (I bumped both numbers that show up)
    DefinitionBlock("DSDT.AM", "DSDT", 0x02, "MSFTVM", "MSFTVM02", 0x00000003)
  f. Modify DSDT.ASL to add the description of the ROSD device within the _SB_ scope.
    Scope(\_SB_)
    {
        Device(ROSD)
        {
            Name(_CID, "RenderOnlySample")
            Name(_HID, "RenderOnlySample")
            Name(_UID, 0x0)
            Name(_CRS, Buffer(0x5)
            {
                0x22,   // IRQ descriptor 
                0x04,   // Low part of IRQ mask, IRQ 2
                0x00,   // High part of IRQ mask
                0x79,   // End tag
                0x00
            })
        }
    }
    0x22, 0x04, 0x00 adds an interrupt resource. 1st byte (0x22) is type of resource.
    0x79, 0x00 indicates end of "Current Resource" _CRS table.
    Reference: http://www.baldwin.cx/~phoenix/reference/docs/acpi_impguide.pdf
  g. Compile the modified DSDT.ASL to generate DSDT.AML
    > asl.exe DSDT.ASL
  f. Load the modified DSDT.AML into the registry for use in next boot
    > asl /loadtable -v DSDT.AML

3. To install the driver, 
  a. Instruct XAML to render with WARP before the driver is fully functional
Windows Registry Editor Version 5.00

[HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\XAML]
"ForceWARP"=dword:00000001

or:

reg.exe add HKLM\SOFTWARE\Microsoft\XAML /v ForceWARP /y REG_DWORD /d 1

  b. Please don't use "Enhanced Session" when running the driver in the VM
  c. devcon.exe is in o:\WDK\Tools\x64\devcon.exe
  d. you only need to copy ros.inf, roskmd.sys and rosumd.dll into a folder and run:
  > devcon install ros.inf ACPI\RENDERONLYSAMPLE
